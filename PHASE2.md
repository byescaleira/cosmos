# Cosmos Phase 2 — Atom Implementation Spec

> Source of record: the 16 verified per-atom specs (JSON) for Phase 2. This document is implementation-ready: an engineer builds each atom from its section below, in the order given in §1, applying the §5 checklist per atom, and resolving the §6 TODOs during implementation.

---

## 1. Overview & Implementation Order

Phase 2 delivers the remaining SwiftUI atoms. Risk is driven by three axes: (a) **style-protocol vs wrap-view** — style-protocol atoms (ButtonStyle/ToggleStyle/LabelStyle/ProgressViewStyle/GroupBoxStyle/MenuStyle) get a clean `makeBody` hook and SE-0299 dot-syntax; wrap-view atoms must accept the native chrome and have no rendering override; (b) **platform guard complexity** — atoms that exist on all 5 platforms need no `#if os()` and rarely need runtime `#available`; atoms absent on tvOS/watchOS need compile guards + fallbacks and the most CI attention; (c) **fragmentation of built-in styles** — Picker/List/TabView expose many styles with wildly different per-platform availability matrices.

### Build order (low-risk style-protocol first, high-risk platform-fragile last)

**Wave A — low-tier, style-protocol, no guard (build confidence + validate the motion/haptics/tracking plumbing).**
1. `CosmosLabel` (LabelStyle) — static display, no motion/haptics; purest style-protocol exercise.
2. `CosmosProgress` (ProgressViewStyle) — passive display, indeterminate-vs-determinate split; validates continuous-loop reduce-motion handling.
3. `CosmosToggle` (ToggleStyle) — first interactive style-protocol atom; validates custom-style a11y re-application + haptics + valueChange motion.

**Wave B — low-tier, wrap-view, no guard (validate the wrap-view pattern + zero-customization discipline).**
4. `CosmosDivider` — leaf primitive, decorative-a11y; simplest wrap-view.
5. `CosmosIcon` (Image) — large modifier surface, no style protocol; validates a11y label vs decorative split.
6. `CosmosLink` — URL-driven, `.openURL` intercept; validates env-based behavior routing.

**Wave C — medium-tier, style-protocol with guards (validate fallback discipline).**
7. `CosmosGroupBox` (`#if !os(tvOS) && !os(watchOS)`) — single built-in style; first fallback view on tvOS/watchOS.
8. `CosmosMenu` (`#if !os(watchOS)`) — MenuStyle conformable; primaryAction + watch fallback.
9. `CosmosDatePicker` (`#if !os(tvOS)`) — DatePickerStyle conformable; severe per-style availability matrix; watchOS floor 10.

**Wave D — medium-tier, opaque style-protocol / wrap-view with guards.**
10. `CosmosTextField` group (TextFieldStyle opaque `_body` + SecureField wrap + TextEditor `#if !os(tvOS) && !os(watchOS)`) — three coordinated atoms; keyboard-modifier `#if os(iOS) || os(tvOS)` guards; `.bordered` iOS 26 gate.
11. `CosmosSlider` (`#if !os(tvOS)`) — no SliderStyle; wrap-view; iOS 26 ticks/neutralValue cluster.
12. `CosmosStepper` (`#if !os(tvOS)`) — no StepperStyle; tvOS Button-pair fallback; native-haptic double-fire risk.

**Wave E — medium-tier, wrap-view, no guard but heavy style fragmentation (highest integration risk).**
13. `CosmosSection` (wrap-view, no guard) — primitive; appearance driven by enclosing List/Form; modifier-guard matrix.
14. `CosmosPicker` (wrap-view, no guard) — PickerStyle is opaque/non-conformable; severe per-style fragmentation; `.tabs` is above Cosmos 26 baseline (do NOT expose).
15. `CosmosList` (wrap-view, no guard) — ListStyle opaque; selection-init watchOS branching; many modifier guards; row-identity discipline.
16. `CosmosTabView` (wrap-view, no guard) — TabViewStyle opaque; modern TabContentBuilder + legacy deprecated inits; per-style `#if os()` guards inside the atom; `.prominent` is OS 27 (exclude).

**Rationale.** Wave A proves the style-protocol plumbing on the easiest atoms; Wave B proves the wrap-view discipline on atoms with no platform branching; Waves C–D introduce `#if os()` guards and runtime `#available` gates one at a time; Wave E is deferred last because the style-fragmentation matrix is the single biggest source of per-platform compile risk and benefits from all the earlier patterns being settled.

## 2. Per-Atom Spec

### 2.1 CosmosLabel

- **Pattern:** style-protocol (`LabelStyle`).
- **Guard:** none. **Runtime `#available`:** none at `.v26` floor (comment the iOS 16/17/26 + iOS 14.5 origins of specific inits/modifiers for future floor-lowering).
- **Inits to expose:** `init(_ titleKey: LocalizedStringKey, systemImage: String)`, `init<S>(_ title: S, systemImage: String)` (S: StringProtocol), `init(_ titleKey: LocalizedStringKey, image: String)`, `init<S>(_ title: S, image: String)`, generic `init(@ViewBuilder title: () -> Title, @ViewBuilder icon: () -> Icon)`, `init(_ titleResource: LocalizedStringResource, systemImage:)` / `(_ titleResource:, image:)` (iOS 16+, primary key/String overloads preferred). Do NOT re-state the title as `accessibilityLabel`.
- **Customization surface:** a `CosmosLabelStyle` conforming to `LabelStyle` composes `configuration.title` and `configuration.icon` (both are `SwiftUICore.View`, Body == Never at the configuration-piece level but usable as Views). Built-ins via `.labelStyle(.automatic/.iconOnly/.titleOnly/.titleAndIcon)`. Modifiers: `.foregroundStyle`, `.font` (relativeTo: on custom fonts), `.symbolEffect` (iOS 17+, on icon only), iOS 26 `.labelReservedIconWidth(_:)` / `.labelIconToTitleSpacing(_:)`.
- **Customization limits:** `configuration.title`/`.icon` are opaque (Body == Never); compose only, never introspect. No pre-iOS-26 control over icon↔title spacing except native layout.
- **Accessibility:** rely on the inferred title as the VoiceOver label (do NOT double-label); allow override via `.cosmosAccessibleLabel`-style modifier only. Dynamic Type flows through title Text automatically.
- **Haptics:** none. Label is static.
- **Motion kind:** `none`. A custom style may apply `.symbolEffect` to the icon (caller-driven); gate on `configuration.motion.isEnabled` only — symbolEffect auto-respects Reduce Motion, do NOT double-gate. Continuous PhaseAnimator on the icon must be suppressed under reduce-motion unless the icon motion is the sole progress signal (`.preserve`).
- **Key modifiers to wire:** `.labelStyle(_:)`, `.foregroundStyle(_:)`, `.font(_:)`, `.labelReservedIconWidth(_:)`, `.labelIconToTitleSpacing(_:)`, `.symbolEffect(_:)`, `.accessibilityLabel/Hint/Identifier`.
- **Open risks / TODOs:** verify iOS 26 `.labelReservedIconWidth`/`.labelIconToTitleSpacing` actually render on watchOS/tvOS small screens (declared available — visually confirm in `#Preview`); `LabelStyle` is `@preconcurrency @MainActor` — keep `CosmosLabelStyle` Sendable and avoid capturing non-Sendable state.

### 2.2 CosmosProgress

- **Pattern:** style-protocol (`ProgressViewStyle`).
- **Guard:** none. **Runtime `#available`:** none at `.v26`.
- **Inits to expose:** indeterminate `init()`, `init(@ViewBuilder label:)`, `init(_ titleKey:)`, `init<S>(_ title:)`; determinate `init<V>(value: V?, total: V = 1.0)` and label/currentValueLabel variants; `init(_ titleKey:, value:, total:)`, `init<S,V>(_ title:, value:, total:)`; `init(_ progress: Progress)`; `init(timerInterval:countsDown:label:currentValueLabel:)` (iOS 16+).
- **Customization surface:** `CosmosProgressStyle : ProgressViewStyle` with `makeBody(configuration:)` reading `fractionCompleted: Double?` (nil = indeterminate), `label`, `currentValueLabel`. Built-ins `.automatic/.linear/.circular` via SE-0299. `.tint(_:)` (Color iOS 15+, ShapeStyle iOS 16+). Custom style can draw its own ring via `Circle().trim(from:to:)`.
- **Customization limits:** built-in style bodies are opaque — no native bar thickness/track color/animation curve tuning without a fully custom `makeBody`. `init(tint:)` on Linear/Circular is **deprecated** — use `.tint(_:)` modifier, never expose a tint init. Indeterminate spinner timing is not exposed.
- **Accessibility:** auto `updatesFrequently` trait (documented, not in interface — treat as plausible-but-unverified). VoiceOver reads percentage (determinate) / "in progress" (indeterminate). Always provide `.accessibilityLabel`; optional `.accessibilityValue` for determinate. A custom `makeBody` MUST re-apply `accessibilityLabel`/`Value`/`traits` (native wiring lost).
- **Haptics:** none. ProgressView is passive; if a completion haptic is wanted, fire from the call site (`.sensoryFeedback(.success)` on completion), gated by `config.haptics` + reduceMotion.
- **Motion kind:** `valueChange` for determinate fills (`.cosmosAnimation(.valueChange, value: fraction)`); `appear`/`disappear` when loading starts/ends. **Indeterminate spinning is continuous looping motion** — suppress under reduce-motion unless progress is the sole signal (`.preserve`); otherwise `.substitute` (opacity crossfade / static "Loading"). Route through `CosmosMotionPolicy.shouldEmit`, NOT bare `@Environment(\.accessibilityReduceMotion)`. When delegating to a built-in style, do NOT double-gate the native spinner (it auto-respects).
- **Key modifiers to wire:** `.progressViewStyle(_:)`, `.tint(_:)`, `.accessibilityLabel/Value`, `.cosmosAnimation(.valueChange, value:)`, `.cosmosMotion(_:)`/`.cosmosMotionTokens(_:)`.
- **Open risks / TODOs:** custom style loses native `updatesFrequently` trait — re-apply; `fractionCompleted` is `Double?` — handle nil as indeterminate, never force-unwrap; `Progress` is `@MainActor`-bound — call-site instances must be MainActor-isolated or value-backed under Swift 6.

### 2.3 CosmosToggle

- **Pattern:** style-protocol (`ToggleStyle`). **Guard:** none. **Runtime `#available`:** none at `.v26` (comment the iOS 16 origin of `isMixed` for floor-lowering).
- **Inits to expose:** `init(isOn: Binding<Bool>, @ViewBuilder label:)` (primary), `init(_ titleKey:, isOn:)`, `init<S>(_ title:, isOn:)`, `init(_ titleResource:, isOn:)` (`@_disfavoredOverload`, iOS 16+), `init(_ titleKey:, systemImage:, isOn:)` and String/resource variants, `init<C>(sources:isOn:label:)` (iOS 16+ multi-row — verify on/off + isMixed per row if exposed), `init(_ configuration: ToggleStyleConfiguration)` for recursive styles.
- **Customization surface:** `CosmosToggleStyle : ToggleStyle` with `makeBody(configuration:)` reading `label`, `$isOn`, `isMixed` (iOS 16+). Built-ins `.automatic/.switch/.button/.checkbox`. `.tint(_:)` (replaces deprecated `SwitchToggleStyle(tint:)`). `.controlSize(_:)`/`.controlSize(_:range)`, `.labelsHidden()`, `.sensoryFeedback(.selection, trigger: isOn)`.
- **Customization limits:** built-in style bodies opaque; no thumb/track/corner-radius tuning without a fully custom style. `.checkbox` macOS-only, `.button` tvOS-unavailable, `.switch` tvOS-min-raised-to-18. `Toggle.body` opaque — cannot wrap and mutate default rendering.
- **Accessibility:** automatic `.isToggle` trait + on/off value are **LOST** when Cosmos renders its own thumb/track — a custom style MUST re-add `.accessibilityAddTraits(.isToggle)` (or `.isButton` for a button-style toggle) and `.accessibilityValue(isOn ? "On" : "Off")` (+ "Mixed" when `isMixed`). Verify with VoiceOver.
- **Haptics:** `.sensoryFeedback(.selection, trigger: isOn)` gated by `CosmosHapticsPolicy` (config.haptics.isEnabled + respectReduceMotion). Coordinate emission with the value-change animation. No-op on macOS gracefully.
- **Motion kind:** `valueChange`. Drive the custom style's thumb position/fill crossfade with a single `withAnimation(theme.motion.spring(for: .valueChange).animation)` per state change; map via `.cosmosAnimation(.valueChange, value: isOn)`. No per-view `.animation(_:value:)` with differing curves (desyncs). The native `SwitchToggleStyle` thumb animation is NOT driven by SwiftUI `Animation` — if delegating to `.switch`, suppress Cosmos motion to avoid double animation.
- **Key modifiers to wire:** `.toggleStyle(_:)`, `.tint(_:)`, `.controlSize(_:)`, `.labelsHidden()`, `.cosmosMotion(_:)`/`.cosmosMotionTokens(_:)`, `.cosmosAnimation(.valueChange, value:)`, `.cosmosContentTransition(_:)`, `.sensoryFeedback(.selection, trigger:)`, `.accessibilityLabel/Value/Hint/Identifier/traits`.
- **Open risks / TODOs:** built-in style platform guards (`.checkbox` `#if os(macOS)`, `.button` `#if !os(tvOS)`, `.switch` requires tvOS ≥ 18 — never lower the tvOS floor below 18); deprecated `SwitchToggleStyle(tint:)` — never expose; verify no double animation when wrapping `.switch` + `.cosmosAnimation`; verify multi-row `sources:` binding semantics if exposed.

### 2.4 CosmosDivider

- **Pattern:** wrap-view (no `DividerStyle`). **Guard:** none. **Runtime `#available`:** none.
- **Inits to expose:** `init()` (forward zero-arg).
- **Customization surface:** `Divider` ignores `.foregroundStyle`/`.background`/`.tint` for the line color — recolor via `.overlay(Color, in: .rectangle)` or replace with a custom `Rectangle`/`Canvas` drawn from theme tokens. Apply theme padding; `.accessibilityHidden(true)`; conditional show/hide via `.cosmosTransition(.sheet)`. Axis is inferred from the enclosing `HStack`/`VStack` — do NOT re-wrap in a container to force an axis.
- **Customization limits:** no style protocol; no color/thickness/dash/axis parameter; recoloring requires an overlay or a hand-drawn `Rectangle`. Decide whether `CosmosDivider` offers a thickness token or a separate `CosmosHairline` view for customized separators.
- **Accessibility:** apply `.accessibilityHidden(true)` — purely decorative, never a VoiceOver focus target. If a themed color overlay is applied, ensure contrast against the container background (gate via `@Environment(\.colorSchemeContrast)` / `@Environment(\.accessibilityDifferentiateWithoutColor)`).
- **Haptics:** none.
- **Motion kind:** `none`. Static. Container-driven motion (list insert/remove) handles conditional appearance: `.cosmosTransition(.sheet)` or `listInsert`/`listRemove` at the container level — one `withAnimation` via `theme.motion.spring(for: .containerTransform)`, not per-divider `.animation(_:value:)`.
- **Key modifiers to wire:** `.accessibilityHidden(true)`, `.overlay(_:in:)`/custom `Rectangle`, `.padding(...)`, `.cosmosTransition(.sheet)`, `.frame(...)` to constrain length.
- **Open risks / TODOs:** confirm overlay covers the hairline reliably (`Rectangle().fill` at the divider's bounds); confirm visionOS availability at runtime (the `@available` line omits visionOS by the pre-visionOS-symbol quirk — should compile/run on visionOS 1.0+, but verify with a visionOS build); verify separator rendering quirks on watchOS/tvOS.

### 2.5 CosmosIcon (Image)

- **Pattern:** wrap-view (no `ImageStyle`). **Guard:** none for core. `#if !os(watchOS)` for `.allowedDynamicRange`/`Image.DynamicRange`. Do NOT expose `init(uiImage:)` (UIKit-bridged, forbidden) or `init(nsImage:)` (AppKit-bridged, exclude for portability). **Runtime `#available`:** none at `.v26` (comment iOS 15/16/17/26 origins of specific modifiers for floor-lowering).
- **Inits to expose:** `init(systemName:)`, `init(systemName:variableValue:)` (iOS 16+), `init(_ name:, bundle:)`, `init(_ name:, bundle:, label: Text)`, `init(decorative name:, bundle:)`, `init(decorative name:, variableValue:, bundle:)`, `init(_ resource: ImageResource)` (iOS 17+), `init(_ cgImage:, scale:, orientation:, label: Text)`, `init(decorative cgImage:, scale:, orientation:)`, `init(size:, label:, opaque:, colorMode:, renderer:)` (iOS 16+).
- **Customization surface:** Image-returning methods: `.resizable(capInsets:resizingMode:)`, `.renderingMode(.template/.original)`, `.interpolation(_:)`, `.antialiased(_:)`, `.symbolRenderingMode(_:)` (iOS 15+), `.symbolVariableValueMode(_:)`/`.symbolColorRenderingMode(_:)` (iOS 26+), `.allowedDynamicRange(_:)` (`#if !os(watchOS)`). View modifiers: `.foregroundStyle`/`.tint` (template symbols only), `.font` + `.imageScale`, `.symbolVariant(_:)`, `.symbolEffect(_:)` (iOS 17+, auto-respects Reduce Motion — gate on `isEnabled` only), `.contentTransition(.symbolEffect(.replace))`, `.scaledToFit/.scaledToFill/.frame/.clipShape/.overlay/.shadow/.grayscale/.opacity`.
- **Customization limits:** raster (asset/CG) images cannot be tinted unless `.renderingMode(.template)` is set first. No `init(uiImage:)`/`init(nsImage:)` in the Cosmos surface. Deprecated `init(_systemName:colorPalette:)` — route to `.renderingMode(.original)`, never wrap.
- **Accessibility:** SF Symbols: VoiceOver may announce the raw symbol name — ALWAYS set `.accessibilityLabel(_:)` explicitly for meaningful symbols and `.accessibilityHidden(true)` for decorative ones. The decorative vs labelled init choice is load-bearing. `.isImage` trait auto-applied.
- **Haptics:** none. If used as a Button/Toggle label, the controlling style owns haptics.
- **Motion kind:** `none`. `.symbolEffect` is caller-driven; a symbol-replace on value change maps to `valueChange` at the use site, not inside the atom.
- **Key modifiers to wire:** `.resizable`, `.renderingMode(.template|.original)`, `.symbolRenderingMode(_:)`, `.symbolEffect(_:options:isActive:)`, `.font(_:)`/`.imageScale(_:)`, `.symbolVariant(_:)`, `.foregroundStyle(_:)`, `.accessibilityLabel(_:)`/`.accessibilityHidden(_:)`, `.cosmosMotion`/`.cosmosAnimation(value:)`/`.cosmosTransition(_:)`.
- **Open risks / TODOs:** SF Symbol a11y — easy to forget `.accessibilityLabel`; document template-mode as the tintable path; `ImageResource` codegen present on all 5 platforms (iOS 17/macOS 14/tvOS 17/watchOS 10 ≤ 26); `.allowedDynamicRange` watchOS-unavailable — verify watchOS compile with the guard.

### 2.6 CosmosLink

- **Pattern:** wrap-view (no `LinkStyle`; `LinkButtonStyle` is a `PrimitiveButtonStyle` for Button, macOS-only — NOT a Link style). **Guard:** none. **Runtime `#available`:** `OpenURLAction.Result.systemAction(_:prefersInApp:)` is iOS 26+ — gate centrally if exposed.
- **Inits to expose:** `init(_ titleKey:, destination:)`, `init<S>(_ title:, destination:)`, `init(_ titleResource:, destination:)` (iOS 16+), `init(destination:, @ViewBuilder label:)`.
- **Customization surface:** generic label (any View). Standard View modifiers on the rendered Link. URL handling interceptable via `.environment(\.openURL, OpenURLAction { ... })` — Cosmos should centralize an openURL intercept for tracking + in-app routing.
- **Customization limits:** no style protocol; no action closure (URL-driven only); no press-state exposure; press/highlight is system-controlled.
- **Accessibility:** system exposes link behavior; set `.accessibilityLabel` when label is iconic, `.accessibilityHint` for destination. Keyboard/focus navigable on tvOS/macOS.
- **Haptics:** none — no observable state change to trigger on. A contrived intercept-counter haptic is not natural; if ever added, gate on `config.haptics.isEnabled && !reduceMotion` with `.selection`.
- **Motion kind:** `none`. Label content may carry its own `symbolEffect` (auto-respects Reduce Motion; gate `isEnabled` only).
- **Key modifiers to wire:** `.environment(\.openURL, ...)` (centralize), `.accessibilityLabel/Hint/Identifier`, `.foregroundStyle(_:)`/`.tint(_:)`, `.font(_:)`, `.cosmosMotion(_:)`/`.cosmosMotionTokens(_:)` (on label content).
- **Open risks / TODOs:** `Link` is `@_Concurrency.MainActor @preconcurrency ~Swift.Sendable` — keep MainActor isolation clean under Swift 6; watchOS openURL may be a no-op/restricted — document; `LinkButtonStyle` must never be wired as a Link style.

### 2.7 CosmosGroupBox

- **Pattern:** style-protocol (`GroupBoxStyle`). **Guard:** `#if !os(tvOS) && !os(watchOS)` (Apple docs incorrectly list tvOS 14+/watchOS 7+ — the SDK marks both unavailable). **Runtime `#available`:** none.
- **Fallback on missing:** on tvOS/watchOS render a plain fallback — the label (as a `Text` header, if any) above the content in a `VStack` with `CosmosTheme` padding/typography, no GroupBox chrome. Keep the public API uniform behind the guard; branch internally.
- **Inits to expose:** `init(@ViewBuilder content:)` (Label == EmptyView), `init(_ titleKey:, @ViewBuilder content:)`, `init<S>(_ title:, @ViewBuilder content:)`, `init(_ titleResource:, @ViewBuilder content:)` (iOS 16+), `init(@ViewBuilder content:, @ViewBuilder label:)` (content-first canonical — NOT the deprecated `init(label:content:)`).
- **Customization surface:** `CosmosGroupBoxStyle : GroupBoxStyle` with `makeBody(configuration:)` receiving `label`/`content` as View-conforming opaque pass-throughs (Body == Never, place-only). Only `DefaultGroupBoxStyle` exists (`.automatic`) — no `.bordered`/`.plain` variants. `.groupBoxStyle(_:)` applies any conforming style.
- **Customization limits:** only one built-in style; its body is an opaque token — any visual deviation requires a full custom `GroupBoxStyle` that reimplements chrome. No `.tint`-style scalar modifier for GroupBox appearance. `.groupBoxStyle` modifier itself is unavailable on tvOS/watchOS.
- **Accessibility:** do NOT force `.isContainer` (would duplicate SwiftUI's grouping). Ensure child atoms carry their own labels/traits. Dynamic Type reflows through label/content (ordinary Views). No `accessibilityIdentifier` on the box itself; rely on child `trackingId` fallback.
- **Haptics:** none — static grouping container.
- **Motion kind:** `none`. A custom style could optionally apply `.cosmosTransition(.containerTransform)` when the box appears/disappears (caller-driven). `matchedGeometryEffect` if used: single `@Namespace`, single `isSource: true`, one `withAnimation(theme.motion.spring(for: .containerTransform).animation)`.
- **Key modifiers to wire:** `.groupBoxStyle(_:)`, `.automatic`, `.cosmosMotion(_:)`/`.cosmosMotionTokens(_:)`/`.cosmosSpringStyle(_:)`, `.cosmosPreviewVariant`/`.cosmosPreviewEnv` (previews).
- **Open risks / TODOs:** confirm the fallback path on tvOS/watchOS compiles without referencing GroupBox symbols; `GroupBoxStyle` is `@MainActor @preconcurrency` — keep `CosmosGroupBoxStyle` `~Swift.Sendable`, makeBody MainActor-isolated; the `init(_ configuration:)` (iOS 14/macOS 11) only needed if a style re-emits a GroupBox from configuration; do not expose the deprecated `init(label:content:)`.

### 2.8 CosmosMenu

- **Pattern:** style-protocol (`MenuStyle`). **Guard:** `#if !os(watchOS)` (Menu unavailable on watchOS). **Runtime `#available`:** none at `.v26` floor (tvOS floor for Menu is 17, below Cosmos 26 — comment for floor-lowering).
- **Fallback on missing:** on watchOS render a `CosmosButton` (or `CosmosPicker` / overflow Sheet) as the substitute overflow surface inside `#if os(watchOS)`; the Menu-backed body lives in `#if !os(watchOS)`.
- **Inits to expose:** `init(@ViewBuilder content:, @ViewBuilder label:)`, `init(_ titleKey:, @ViewBuilder content:)`, `init(_ titleResource:, @ViewBuilder content:)`, `init<S>(_ title:, @ViewBuilder content:)`, `init(_ titleKey:, systemImage:, @ViewBuilder content:)`, `init(_ titleResource:, systemImage:, @ViewBuilder content:)`, `init<S>(_ title:, systemImage:, @ViewBuilder content:)`, `init(_ titleKey:, image: ImageResource, @ViewBuilder content:)` (iOS 17+), `init(..., primaryAction:)` variants (iOS 15+), `init(_ configuration: MenuStyleConfiguration)`.
- **Customization surface:** `CosmosMenuStyle : MenuStyle` with `makeBody(configuration:)` — fully customize the TRIGGER (wrap `configuration.label` with `.buttonStyle(.bordered/.borderless/.plain/.glass/.glassProminent)`, `.tint`, `.controlSize`, `.labelStyle`, `.font`, padding, `.glassEffect`, a11y). Built-ins `.automatic` (DefaultMenuStyle) and `.button` (ButtonMenuStyle, iOS 16+). Modifiers: `.menuOrder(_:)` (`.priority` unavailable on macOS/tvOS/watchOS — iOS/visionOS only), `.menuIndicator(_:)` (tvOS 17+ floor, watchOS unavailable), `.menuActionDismissBehavior(_:)` (`.disabled` available iOS 16.4+/tvOS 17.0+/visionOS 1.0+, unavailable macOS/watchOS — NOT tvOS-exclusive), `.buttonStyle(_:)` paired with `.menuStyle(.button)`. Content builder accepts Button, Toggle, Picker(.menu), nested Menu, Section, Divider, ControlGroup.
- **Customization limits:** popover content list is opaque (`MenuStyleConfiguration.Content`/`.Label` Body == Never) — cannot inspect/decompose/re-render popover rows/separators/chrome/highlight/surface; only the trigger's surrounding chrome is customizable. `BorderedButtonMenuStyle` macOS-only deprecated; `BorderlessButtonMenuStyle` deprecated — route to `.menuStyle(.button)` + `.buttonStyle(.bordered/.borderless)`. watchOS has no Menu. Popover presentation + Liquid Glass chrome at OS 26 are system-controlled.
- **Accessibility:** trigger needs descriptive `accessibilityLabel` (Text/Label title suffices; set explicitly when icon-only). Set `accessibilityIdentifier` on the trigger for tracking. Contained Buttons/Toggles inherit standard traits.
- **Haptics:** plain (no-primary) Menu — none on open (Apple menus don't haptic on open; contained Buttons own theirs). primaryAction Menu — `.sensoryFeedback(.selection, trigger:)` (or `.impact(.rigid)` if destructive) on primary tap. Gate through `CosmosHapticsPolicy.shouldEmit`.
- **Motion kind:** `press` — the trigger press response. Popover appear/dismiss is native system-controlled — do NOT map `.sheet`/`.appear`/`.containerTransform` to the popover. Single-resolver rule: Cosmos owns only the trigger.
- **Key modifiers to wire:** `.menuStyle(_:)`, `.buttonStyle(_:)` (with `.menuStyle(.button)`), `.controlSize(_:)`/`.labelStyle(_:)`/`.font(_:)`/`.tint(_:)`, `.menuOrder(_:)`, `.menuIndicator(_:)`, `.menuActionDismissBehavior(_:)`, `.accessibilityLabel/Value/Hint/Identifier/traits`, `.cosmosAnimation(.press, value:)`, `.cosmosMotion(_:)`/`.cosmosMotionTokens(_:)`/`.cosmosSpringStyle(_:)`, `.cosmosTrackingId(_:)`.
- **Open risks / TODOs:** `MenuStyle` is `@preconcurrency @MainActor` — zero Swift 6 warnings (mark conformance `@preconcurrency` if needed, struct stays Sendable); configuration.Label/Content opaque — never decompose; verify `primaryAction` closure is `@Sendable`-friendly; verify `.glassProminent` availability on the target; do NOT apply `.glassEffect` to popover content.

### 2.9 CosmosDatePicker

- **Pattern:** style-protocol (`DatePickerStyle`). **Guard:** `#if !os(tvOS)` (DatePicker is `@available(tvOS, unavailable)` at the type level — cannot even be referenced on tvOS). **Runtime `#available`:** none at `.v26` (watchOS floor is 10.0; `.hourMinuteAndSecond` is watchOS-exclusive — `#if os(watchOS)` guard, not merely `#available`).
- **Fallback on missing:** tvOS — compile-time absence; app-level code chooses an alternative (CosmosPicker or custom navigation-based date chooser). No in-place fallback.
- **Inits to expose:** `init(_ titleKey:, selection:, displayedComponents:)`, `init(_ titleKey:, selection:, in: ClosedRange<Date>, displayedComponents:)`, PartialRangeFrom/PartialRangeThrough variants, `init<S>(_ title:, selection:, displayedComponents:)`, `init(selection:, displayedComponents:, @ViewBuilder label:)` and range variants, `init(_ titleResource:, selection:, displayedComponents:)` (iOS 16+).
- **Customization surface:** `CosmosDatePickerStyle : DatePickerStyle` with `makeBody(configuration:)` reading `label`, `$selection`, `minimumDate`, `maximumDate`, `displayedComponents`. Six built-in styles via SE-0299: `.automatic`, `.wheel` (iOS/watchOS; NOT macOS), `.graphical` (iOS/macOS; NOT watchOS/tvOS), `.compact` (iOS/macOS 10.15.4; NOT watchOS/tvOS), `.field`/`.stepperField` (macOS-only). `.datePickerStyle(_:)` (tvOS unavailable). Range constraints via `in:`. Custom Label via `@ViewBuilder`.
- **Customization limits:** inner rendering of wheel/graphical/compact is opaque/native-bridged. No control over wheel-row text color, calendar day-tint, popover chrome, compact-display text color. Field/StepperField macOS-only; Wheel macOS-unavailable; Graphical/Compact watchOS/tvOS-unavailable. No clear-button/placeholder. `.hourMinuteAndSecond` watchOS-only.
- **Accessibility:** label is the VoiceOver label; value is system-announced. Wheel/graphical expose adjustable-like interaction. Set `.accessibilityHint` when constraining to a range. `accessibilityIdentifier` for tracking.
- **Haptics:** `.sensoryFeedback(.selection, trigger: selection)` — DatePicker does NOT emit a system haptic automatically (unlike Toggle/Slider), so this is additive. ### 2.16 CosmosTabView

- **Pattern:** wrap-view (`TabViewStyle` is NOT meaningfully conformable — only underscored `_makeView`/`_makeViewList`; Cosmos must wrap `TabView` and apply built-in styles only). **Guard:** none at the atom level; per-style `#if os()` guards inside the atom. **Runtime `#available`:** none at `.v26` for base; `tabBarMinimizeBehavior` all-5 iOS 26; `tabViewBottomAccessory` iOS 26.0/26.1 (iOS-only); `TabRole.prominent` is `anyAppleOS 27.0` — NOT at Cosmos 26 baseline (omit entirely).

Do NOT use the deprecated `init(selection: Binding?, @ViewBuilder content:)` or `.tabItem { }` + `.tag(_)` (deprecated 100000.0 — warnings-as-failures).
- **Customization surface:** `.tabViewStyle(_:)` with built-ins `.automatic` (DefaultTabViewStyle, all platforms iOS 14+), `.page` (PageTabViewStyle, iOS/tvOS/watchOS — NOT macOS), `.sidebarAdaptable` (iOS 18/macOS 15/tvOS 18/visionOS 2 — NOT watchOS), `.tabBarOnly` (same as sidebarAdaptable — NOT watchOS), `.verticalPage` (VerticalPageTabViewStyle, watchOS 10 only — renamed from deprecated CarouselTabViewStyle), `.grouped` (GroupedTabViewStyle, macOS 15 only). No custom tab bar layout / custom selection indicator / custom More-overflow without UIKit. `TabRole.prominent` is OS 27 (exclude).
- **Accessibility:** each Tab/TabItem label auto-exposes its accessibility label; tab bar items get button trait natively. VoiceOver announces focused tab name + selection changes. For custom label content, apply explicit `.accessibilityLabel`/`.accessibilityValue`. Dynamic Type scales tab labels; page-index dots do not scale. tvOS tabs are focus-engine driven. `.accessibilityIdentifier` per tab for tracking.
- **Haptics:** `.sensoryFeedback(.selection, trigger: selectionValue)` on tab change — gated via `cosmosConfiguration.haptics.enabled AND accessibilityReduceMotion`. No `.impact` needed (discrete value change, not a step). No-op where no hardware.

- **Open risks / TODOs:** per-style platform divergence is the biggest risk — each needs a compile-time `#if os(...)` guard inside CosmosTabView, not a single atom-level guard (PageTabViewStyle macOS-unavailable; SidebarAdaptable/TabBarOnly watchOS-unavailable but available on tvOS 18; VerticalPageTabViewStyle watchOS-only; GroupedTabViewStyle macOS-only; TabViewCustomization tvOS+watchOS-unavailable; tabViewBottomAccessory iOS-only); `TabViewStyle` not conformable — wrap, do NOT attempt a custom `CosmosTabViewStyle`; legacy `init(selection: Binding?, @ViewBuilder content:)` + `.tabItem` deprecated — using them violates zero-warnings; `CarouselTabViewStyle` deprecated/renamed to `VerticalPageTabViewStyle` — never reference Carousel; `TabRole.prominent` OS 27 — omit entirely from Cosmos 26; `tabBarMinimizeBehavior` all-5 at iOS 26 — verify no-op/ignored on tvOS/watchOS where there's no minimizable bar; verify `TabContentBuilder` init compiles cleanly on all 5 with zero concurrency warnings (`TabView` is `~Swift.Sendable`; `TabContent` is `@preconcurrency @MainActor`).

## 5. Cross-Cutting Integration Checklist (apply per atom)

For each atom, before marking it complete, verify all 11 items:

1. **Accessibility** — label/value/hint/identifier/traits/customContent as appropriate; gates `reduceMotion`/`reduceTransparency`/`colorSchemeContrast`/`differentiateWithoutColor` read through `cosmosConfiguration` (not bare env); Dynamic Type reflow (custom fonts pass `relativeTo:`); for custom style-protocol `makeBody` that reimplements rendering, RE-APPLY lost native traits/labels (Toggle `.isToggle`+value; ProgressView `updatesFrequently`+label/value; Menu trigger label; GroupBox children carry their own).
2. **Haptics** — `.sensoryFeedback` gated by `CosmosHapticsPolicy.shouldEmit(isEnabled:respectReduceMotion:reduceMotion:)` (NOT bare env); no-op where no hardware; coordinate with value-change animation; do NOT double-haptic against native emissions (Stepper, Toggle delegated to `.switch`).
3. **Localization** — strings via `CosmosLocalizationConfiguration`; prefer `LocalizedStringKey`/`LocalizedStringResource` inits; public string-constant symbols; baseline `en` + `pt-BR`.
4. **Tracking** — `CosmosTrackingConfiguration.track(_:)` with `componentId = trackingId ?? accessibilityIdentifier`; opt-in, passive, no network/PII.
5. **Motion** — `.cosmosAnimation(_ kind:, value:)`/`.cosmosTransition(_:)`/`.cosmosContentTransition(_:)` resolved through `CosmosMotionTokens.animation(for:)`; gated by `CosmosMotionPolicy.shouldEmit` (config-aware, NOT bare `accessibilityReduceMotion`); `symbolEffect` gated on `isEnabled` ONLY (auto-respects RM — do NOT double-gate); springs preferred; single `withAnimation` per coordinated state change; no per-view `.animation(_:value:)` with differing curves.
6. **Enable** — `cosmosConfiguration.enable` (or relevant config flag) gates the atom's active behavior; disabled state is visually coherent.
7. **Loading** — where relevant (ProgressView, Picker/Toggle/Menu in loading contexts), `cosmosConfiguration.loading` reflects loading state.
8. **Log** — `cosmosConfiguration.log` for diagnostic events (no PII, no network).
9. **Error** — `cosmosConfiguration.error` for error reporting (e.g. binding validation failures).
10. **Layout-adaptive** — `AnyLayout`/`ViewThatFits` switched by `horizontalSizeClass`/`verticalSizeClass`/`dynamicTypeSize` so view identity (focus/scroll/animation state) survives reflow; no `if/else` that recreates view identity on rotation. Verify in landscape + Dynamic Type accessibility previews.
11. **Multiplatform** — correct `#if os()` guards per §4; build for EACH target platform (iOS/macOS/tvOS/watchOS/visionOS) to confirm `#if os()` coverage; runtime `#available` centralized; `#Preview` blocks per variant (default / disabled-loading / dark / Dynamic Type accessibility / landscape / RTL / per-platform) using `#Preview(_:traits:)` — NOT the deprecated `.previewDevice`/`.previewLayout`/etc.; inject env via `.cosmosPreviewEnv(...)`/`.cosmosPreviewVariant(_:)` and accessibility SPI keys (`._accessibilityReduceMotion`, etc.); shared setup via `CosmosPreviewModifier`.
