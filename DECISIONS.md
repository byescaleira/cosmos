# Decisions

| Date | Decision | Context | Consequences | Status |
|------|----------|---------|--------------|--------|
| 2026-06-21 | Codename `Cosmos` | Reserved design system codename per byescaleira skill; represents the v26 design system. | Repository `byescaleira/cosmos`, all symbols prefixed with `Cosmos`. | Decided |
| 2026-06-21 | Foundation-first approach | Previous attempt created components before the base, causing inconsistency. | Removed atoms/molecules/organisms; built `CosmosBase` first. | Decided |
| 2026-06-21 | `CosmosConfiguration` as value type | `@Observable` + `@MainActor` caused Swift 6 concurrency friction in tests. | Configuration is a `struct`; mutations happen by replacement via environment or `@State`. | Decided |
| 2026-06-21 | Six base contracts | Components need accessibility, localization, log, error, loading, enablement. | `CosmosBase` contains `Accessibility`, `Localization`, `Log`, `Error`, `Loading`, `Enable` sub-modules. | Decided |
| 2026-06-21 | Swift tools version 6.2 | Required for Apple v26 platform version APIs and SwiftUI `@Entry`. | Toolchain must support Swift 6.2; older Xcode cannot open the package. | Decided |
| 2026-06-21 | Split `CosmosConfiguration` from `CosmosTheme` | Previous `CosmosTheme` mixed behavior (isEnabled, isLoading) with visual tokens. | `CosmosConfiguration` owns behavior contracts; `CosmosTheme` owns visual tokens. Both are environment values. | Decided |
| 2026-06-21 | Adopt SwiftUI `@Entry` for environment values | Manual `EnvironmentKey` is no longer the recommended pattern in v26 SDKs. | `CosmosEnvironment.swift` declares `@Entry` for `cosmosConfiguration` and `cosmosTheme`; generated modifiers replace hand-written ones. | Decided |
| 2026-06-22 | Semantic token layer for theme | Raw fonts, colors, and points make multi-platform theming fragile. | `CosmosTheme` carries `CosmosColorTokens`, `CosmosTypographyTokens`, `CosmosSpacingTokens`, and `CosmosRadiusTokens`. Platform overrides can be added later without API breakage. | Decided |
| 2026-06-22 | Minimal atom API: content in init, state in environment | `CosmosButton` and other atoms mixed `isEnabled`, `isLoading`, `log`, `error`, and accessibility parameters with content in their initializers. | Atoms accept only content via `init`; state/config/theme come from the environment. Added `.cosmosEnabled`, `.cosmosVisible`, `.cosmosReadOnly`, `.cosmosLoading`, `.cosmosAccessibilityLabel`, `.cosmosAccessibilityHint`, `.cosmosAccessibilityHidden`, `.cosmosAccessibilitySortPriority` modifiers that re-inject a mutated `CosmosConfiguration`. | Decided |
| 2026-06-22 | Flat atom folder structure | Per-component subfolders added friction without adding value for atoms. | All atoms live directly in `Sources/Cosmos/Atoms/`. | Decided |
| 2026-06-22 | Data-driven screen renderer | Need to assemble screens from API/JSON payloads using the same atoms. | Added `CosmosScreen` target with `CosmosComponent`, `CosmosScreenRenderer`, `CosmosActionRegistry`, and `CosmosScreenLoader`. | Decided |
| 2026-06-22 | Custom `CosmosComponent` JSON envelope | Swift's synthesized `Codable` for enums produces `{"text":{"_0":...}}`, which is awkward for JSON APIs. | Added manual `Codable` conformance with keyed envelopes (`{"text":{"content_key":"..."}}`) and snake_case support. | Decided |
| 2026-06-22 | ViewInspector + SnapshotTesting for UI tests | ViewInspector gives fast headless structural coverage; SnapshotTesting catches visual/state regressions on iOS. | `CosmosUITests` target split into `ViewInspector/` and `Snapshot/`. Snapshot tests use `UIHostingController` under `#if canImport(UIKit)`. | Decided |
| 2026-06-22 | Remove empty resource bundle | Empty `Sources/Cosmos/Resources` folder caused Xcode codesign failure when running on iOS simulator. | Removed `.process("Resources")` and empty directory; `Bundle.module` usage in tests updated to `Bundle.main`. | Decided |
| 2026-06-22 | Accept SwiftUI environment warning under ViewInspector | Custom `@Entry` environment values read outside an installed view hierarchy log a warning but behave correctly at runtime and in `UIHostingController` snapshots. | No code change; warning is documented as known limitation of headless inspection. | Decided |
| 2026-06-22 | Resolve strings through localization configuration | Atoms accepted raw `String` and rendered JSON keys literally. Localization contract existed but was not consumed. | `CosmosText` resolves its content via `configuration.localization.string(for:)`. `CosmosButton` title uses the same mechanism through `Text(titleKey)` inside the environment. | Decided |
| 2026-06-22 | Conditional accessibility label/hint | Applying `.accessibilityLabel(Text(""))` when no override was configured silenced VoiceOver fallback. | Added `accessibilityLabelOrNil(_:)` and `accessibilityHintOrNil(_:)` helpers in `CosmosBase`; atoms only apply overrides when non-nil. | Decided |
| 2026-06-22 | Decouple `CosmosScreen` models from `Cosmos` | `CosmosLayout.swift` and `CosmosComponentModels.swift` imported `Cosmos` only for `CosmosPadding`, which lives in `CosmosBase`. | Models now import `CosmosBase`; `CosmosScreen` target declares explicit dependency on `CosmosBase`. | Decided |
| 2026-06-22 | Harden atoms before adding molecules | Existing atoms needed more native controls and better test coverage before expanding to molecules. | Added `lineLimit`, `alignment`, `truncationMode`, and `verbatim` to `CosmosText`; `controlSize` to `CosmosButton`; `resizable`, `renderingMode`, and `aspectRatio` to `CosmosIcon`; `dividerThickness` to `CosmosDivider`; all atoms now support `.cosmosRedacted`. | Decided |
| 2026-06-22 | Add redaction as a base contract | Skeleton/loading states are a cross-cutting concern like loading and enablement. | Added `CosmosRedactionConfiguration` and `.cosmosRedacted(_:)` modifier. | Decided |
| 2026-06-22 | Add `CosmosControlSize` theme selector | SwiftUI `ControlSize` is iOS 15+ and useful for buttons, but theme files should not depend on SwiftUI symbols. | Added `CosmosControlSize` enum with `swiftUIValue` mapping and `.cosmosControlSize(_:)` modifier. | Decided |
| 2026-06-22 | Exclude snapshot baselines from SwiftPM target | Snapshot PNGs inside `CosmosUITests` triggered "unhandled resource" warnings. | Added `exclude: ["Snapshot/__Snapshots__"]` to `CosmosUITests` in `Package.swift`. | Decided |
| 2026-06-22 | Add `CosmosImage`, `CosmosLabel`, and `CosmosSpacer` atoms | User requested more atoms; `CosmosImage` needs resource, SF Symbol, and URL support with cache, placeholder, and loading redaction. | `CosmosImage` supports multiple sources and uses `AsyncImage` for remote URLs, sharing the system cache; `CosmosLabel` composes `CosmosIcon` and `CosmosText`; `CosmosSpacer` wraps SwiftUI `Spacer`. | Decided |
| 2026-06-22 | Add `CosmosLink`, `CosmosTextField`, `CosmosToggle`, `CosmosProgress`, `CosmosSlider`, `CosmosPicker`, and `CosmosBadge` atoms | Need to round out the atom library before molecules. | `CosmosLink` uses SwiftUI `Link`; `CosmosTextField`/`CosmosToggle`/`CosmosSlider`/`CosmosPicker` accept `Binding` values because editable state must be caller-owned; `CosmosProgress` uses `ProgressView` for spinner and bar; `CosmosBadge` uses `Text`/Shape with theme variants. All still read enabled/visible/redacted state from `CosmosConfiguration`. | Decided |
| 2026-06-21 | Plugin `/byescaleira` in Claude Code | User wants Claude Code + Ollama + byescaleira plugin to generate Cosmos code. | Will create plugin with project context and guardrails. | In progress |
| 2026-06-22 | Add `CosmosStepper`, `CosmosDatePicker`, and `CosmosMenu` atoms | Final batch of atoms before molecules. | `CosmosStepper` uses SwiftUI `Stepper`; `CosmosDatePicker` uses SwiftUI `DatePicker`; `CosmosMenu` uses SwiftUI `Menu`. All accept required `Binding` values where state is caller-owned and respect `.cosmosEnabled` / `.cosmosRedacted` from environment. | Decided |
| 2026-06-22 | Cross-platform `DatePickerComponents` in `CosmosScreen` models | `CosmosScreen` must not depend on SwiftUI, but `DatePicker` needs `displayedComponents`. | Added a serializable `DatePickerComponents` enum to `CosmosDatePickerModel` and a renderer mapping to SwiftUI `DatePicker.Components`. | Decided |
| 2026-06-22 | Container atoms (`CosmosList`, `CosmosSection`, `CosmosTabView`) are atoms, not organisms | They expose single composable concepts and fit the flat atom folder. | Kept the flat atom folder; organisms reserved for later. | Decided |
| 2026-06-22 | Pure-SwiftUI adaptive container strategy | iOS 27 makes iPhone apps resizable; Apple recommends size classes over orientation/idiom. | `CosmosTabView` switches between `TabView` (compact) and `NavigationSplitView` sidebar (regular) using `horizontalSizeClass`. | Decided |
| 2026-06-22 | Defer UIKit `UITabBarController.sidebar` bridge | UIKit's sidebar API is iOS 27-only and would introduce a UIKit dependency. | Stay native-first with SwiftUI; revisit UIKit bridge if pure-SwiftUI strategy falls short. | Decided |
| 2026-06-22 | Platform-gated list styles | `.grouped` and `.insetGrouped` are unavailable on macOS. | `CosmosListStyle` falls back to `.automatic` on unsupported platforms. | Decided |
| 2026-06-22 | Explicit `CodingKeys` for acronym JSON properties | `JSONEncoder` snake_case produces `selected_tab_id`, but `JSONDecoder` reads it back as `selectedTabId`, breaking round-trips. | Models use explicit `CodingKeys` with raw values like `"selectedTabId"` and `"selectedItemIds"`. | Decided |
| 2026-06-22 | Keep project context for Claude Code in-repo under `.claude/` | The global `byescaleira-plugin` provides identity and rules, but Cosmos needs project-specific architecture and atom context. | Added `.claude/CLAUDE.md`, `.claude/commands/byescaleira.md`, and `.claude/skills/cosmos/SKILL.md` inside the Cosmos repository. | Decided |
| 2026-06-22 | Molecule layer lives in `Sources/Cosmos/Molecules/` | Atoms are in `Sources/Cosmos/Atoms/`. Molecules are small atom groups with recognizable UI patterns. | Created `Sources/Cosmos/Molecules/` for `CosmosInputRow`, `CosmosListRow`, `CosmosFormRow`. | Active |
| 2026-06-22 | Interactive molecules use renderer-local `@State` + action dispatch | JSON cannot carry runtime `Binding`. | `CosmosScreenRenderer` wraps interactive molecules in local `@State` wrappers and dispatches actions on changes. | Active |
| 2026-06-22 | First batch of molecules: `CosmosInputRow`, `CosmosListRow`, `CosmosFormRow` | These three patterns unlock forms, lists, and settings screens. | Shipped with `CosmosScreen` JSON support and renderer wrappers. | Decided |
| 2026-06-22 | Second batch of molecules: `CosmosEmptyState`, `CosmosButtonRow`, `CosmosSearchBar` | Empty states, CTAs, and search are universal iOS patterns. | Shipped with `CosmosScreen` JSON support and renderer wrappers. | Decided |
| 2026-06-22 | Third batch of molecules: `CosmosStatusRow`, `CosmosCard` | Status rows and cards are common in feeds, notifications, and dashboards. | `CosmosStatusRow` reuses `CosmosImage` source switching like `CosmosEmptyState`; `CosmosCard` uses surface background and medium corner radius. | Decided |
| 2026-06-22 | Fourth batch of molecules: `CosmosAlertBanner`, `CosmosLoadingState` | Inline alerts and loading states complete the standard molecule set. | `CosmosAlertBanner` maps variants to theme semantic colors; `CosmosLoadingState` composes `CosmosProgress` with optional title/subtitle. | Decided |
