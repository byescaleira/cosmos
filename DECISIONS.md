# Decisions

Architecture decisions for the **Cosmos 26 rebuild** (from-scratch, starting 2026-07-16).
These supersede the pre-rebuild decision log. See `CLAUDE.md` for the binding guidelines and `VERSIONING.md` for the versioning policy.

| Date | Decision | Context | Consequences | Status |
|------|----------|---------|--------------|--------|
| 2026-07-16 | From-scratch rebuild with git history reset | Previous structure accumulated inconsistency (mixed targets, snapshot/ViewInspector tests, single-platform). User wants a clean baseline. | Repo wiped to a single initial commit (with backup branch/tag/stash). All files re-authored fresh; proven foundations may be mirrored. | Decided |
| 2026-07-16 | Multiplatform — all 5 platforms at `.v26` | User wants every component to respond best per platform on iOS/macOS/tvOS/watchOS/visionOS 26. | `Package.swift` targets all 5 at `.v26`; `swift-tools-version: 6.4`; platform-absent components gated with `#if os()`. | Decided |
| 2026-07-16 | UIKit-free package | No `import UIKit`/`UIColor`/`UIViewController`/`#if canImport(UIKit)`/`UIHostingController`. SwiftUI APIs that wrap UIKit internally are acceptable. | Fonts via CoreText (`CTFontManagerRegisterFontsForURL`); haptics via `.sensoryFeedback`; system colors via `Color(.systemBackground)`/`Color(nsColor:)`. | Decided |
| 2026-07-16 | Global state & theme via `@Entry` environment (not per-component) | User: "um componente deve ter seu state, seu theme … tudo global" — concerns are global, each atom reads its relevant subset. | No per-component state/theme structs. `cosmosConfiguration` (behavior) + `cosmosTheme` (visual) + `cosmosTrackingId` injected via `@Entry`. Per-instance overrides via `.cosmos*` modifiers that re-inject a mutated copy. | Decided |
| 2026-07-16 | Split behavior (`CosmosConfiguration`) from visual (`CosmosTheme`) | Mixing `isEnabled`/`isLoading` with color tokens conflated concerns. | `CosmosConfiguration` aggregates 9 behavior contracts; `CosmosTheme` carries tokens + default selectors + `version`. Both are environment values. | Decided |
| 2026-07-16 | 9 cross-cutting contracts | Every component needs accessibility, localization, log, error, loading, enable, plus **haptics**, **tracking**, and **motion** (new). | `CosmosAccessibility`, `CosmosLocalization`, `CosmosLog`, `CosmosError`, `CosmosLoading`, `CosmosEnable`, `CosmosHaptics`, `CosmosTracking`, `CosmosMotion`. All `Sendable`; handlers `@Sendable`. | Decided |
| 2026-07-16 | Modern Swift 6 concurrency — zero warnings, no NSLock | User: "não vem com nslock … tudo conforme a nova concorrência, sem warnings." Swift 6.4 / mode v6 / Xcode 26. | One-time work uses the **once-token `static let`** pattern (`swift_once`); mutable flags use `Mutex`/`Atomic` from `import Synchronization`. All public value types `Sendable`; handlers `@Sendable`; runtime theme `@Observable @MainActor`; `@Entry` values `Sendable`. No `nonisolated(unsafe)` mutable globals. | Decided |
| 2026-07-16 | Runtime theme via `@Observable @MainActor` | Live theme switching needs mutable state; immutable `Sendable` struct is the primary path. | `CosmosThemeObservable` wraps a `CosmosTheme`; injected so access stays main-actor-isolated. (Exact injection pattern confirmed against Swift 6 guidance.) | Decided |
| 2026-07-16 | Versioning: Cosmos N ↔ OS N (baseline Cosmos 26) | User: "totalmente versionado assim como SwiftUI … relacionado à versão do iOS, vamos começar com o cosmos 26." | Library major == OS major; `@available(iOS 26,*)` is API versioning; `if #available` gates centralized; `CosmosVersion` runtime design-language pin in theme; semver minor/patch within major; deprecation runway. See `VERSIONING.md`. | Decided |
| 2026-07-16 | Localization: String Catalogs, modern stack, no plugin | User rejected `Bundle.module` string-table plumbing; `.xcstrings` is current (WWDC25). | `Package.swift` uses `.process("Resources")`; resolve via `LocalizedStringResource`/`#bundle`/`String(localized:)` + public string constants. Baseline `en` + `pt-BR`, extensible. | Decided |
| 2026-07-16 | Layout: `AnyLayout`/`ViewThatFits` for portrait↔landscape | Apple is adapting portrait apps to landscape; reflow must preserve view identity. | Atoms switch layout by `horizontalSizeClass`/`verticalSizeClass`/`dynamicTypeSize` without `if/else` that recreates identity. | Decided |
| 2026-07-16 | Style protocol where conformable, wrap `View` otherwise | Honors each component's real customization surface. | `ButtonStyle`/`ToggleStyle`/`LabelStyle`/`ProgressViewStyle`/`GroupBoxStyle`/`MenuStyle` + SE-0299 dot-syntax; Slider/Stepper/TextField/DatePicker/Picker/List/Section/TabView/Divider/Image/Link/Spacer wrap a `View`. | Decided |
| 2026-07-16 | Custom fonts via `Font.custom(_:size:relativeTo:)` | Custom fonts must scale with Dynamic Type, including accessibility sizes. | `CosmosTypographyTokens.font(for:)` always passes `relativeTo:`. Cosmos ships **no bundled fonts** (system font by default); consumers register custom fonts in their app (`Info.plist` `UIAppFonts` or `CTFontManagerRegisterFontsForURL`) and pass the PostScript name via `.cosmosFont(_:for:)`. | Decided |
| 2026-07-16 | Semantic token layer | Raw points/colors/fonts make multiplatform theming fragile. | `CosmosColorTokens` (platform-conditional, high-contrast adaptive), `CosmosTypographyTokens`, `CosmosSpacingTokens` (4-pt grid), `CosmosRadiusTokens`; selectors `CosmosTextStyle`/`CosmosPadding`/`CosmosButtonStyle`/`CosmosControlSize`. | Decided |
| 2026-07-16 | Liquid Glass: `.glass` selector uses native styles, chrome for the rest | `.glass`/`.glassProminent` are not customizable via `ButtonStyle`. | `CosmosButtonStyle.glass` → `CosmosButton` applies `.buttonStyle(.glassProminent)` (iOS 26 gate) directly; `primary`/`secondary`/`danger`/`ghost` render via `CosmosButtonChrome`. | Decided |
| 2026-07-17 | Motion as the 9th cross-cutting contract (`CosmosMotionConfiguration`) + visual tokens (`CosmosMotionTokens` in `CosmosTheme`) | Motion is a tokenized cross-cutting concern in every mature design system (Material 3, Carbon, Polaris, Lightning, Atlassian); Apple HIG + WWDC23 mandate spring-first, purposeful, optional, reduce-motion-aware motion. Must fit the existing global-state architecture with zero structural deviation. | `CosmosMotionConfiguration` (behavior: isEnabled/respectReduceMotion/reduceMotionPolicy/respectReduceTransparency/handler @Sendable) added as the 9th member of `CosmosConfiguration` — mirrors `CosmosHapticsConfiguration` line-for-line. `CosmosMotionTokens` (visual: CosmosSpring presets, CosmosDuration scale, CosmosTransition/ContentTransition presets + the single `animation(for:reduceMotion:policy:)` resolver) added to `CosmosTheme` — mirrors `CosmosTypographyTokens`. Atoms call `.cosmosAnimation(.press, value:)`/`.cosmosTransition(.sheet)`/`.cosmosContentTransition(.numeric)` (the chokepoint); never raw `.animation(_:value:)`/`.transition(_:)`. `CosmosMotionPolicy.shouldEmit` gates reduce-motion config-aware (not the bare env value), mirroring `CosmosHapticsPolicy`. No `if #available` gating at the `.v26` baseline — every motion primitive (Spring/PhaseAnimator/KeyframeAnimator/blurReplace/symbolEffect/withAnimation(completion)/transition<T>/matchedGeometryEffect) is iOS 17/18 ≤ 26 on all 5 platforms. | Decided |
| 2026-07-17 | Reduce-motion policy: `.substitute` (opacity crossfade) as default, `.instant` (snap), `.preserve` (WCAG 2.3.3 exempt) | Reduce-motion ≠ no feedback (MDN/MFA11y/Verdigris): spatial movement is suppressed but opacity crossfades and color transitions are kept (vestibular-safe, still signal change). The "kill switch" anti-pattern strips feedback. | `CosmosReduceMotionPolicy` enum on `CosmosMotionConfiguration` (default `.substitute`). `CosmosMotionPolicy.transition(full:substitute:reduceMotion:policy:)` picks the variant. Symbol effects auto-respect reduce-motion and are gated on `isEnabled` only (not double-gated) — documented to prevent silencing. Continuous/looping motion suppressed unless sole progress signal (`.preserve`). | Decided |
| 2026-07-17 | Spring presets mirror SwiftUI built-ins + two Cosmos extensions | Polaris removed overshoot/anticipate for minimal production usage — over-shipping presets is an anti-pattern. Apple HIG warns against gratuitous motion. | Five `CosmosSpring` presets: `.cosmosSmooth/.cosmosSnappy/.cosmosBouncy` (mirror SwiftUI's `.smooth/.snappy/.bouncy`) + `.cosmosGentle` (large/visionOS-safe relocations, no overshoot) + `.cosmosInteractive` (gesture velocity preservation). `CosmosSpringStyle` selector + SE-0299 dot-syntax, paralleling `CosmosButtonStyle`. Six-tier `CosmosDuration` scale (Carbon-hybrid: instant/70/110/150/240/400/700ms). | Decided |
| 2026-07-17 | Preview/mock infrastructure in the main `Cosmos` target, public, no `#if DEBUG`, hand-rolled seeded RNG | Consumer apps reuse `CosmosMock` in their own previews/tests; a separate preview target would force a second product/dependency edge for no gain (mock generators are tiny, Sendable, no-UIKit, multiplatform). Zero-third-party-deps rule rules out SwiftFixtures/Genything/LoremSwiftify. | `Sources/Cosmos/Base/Preview/` hosts `CosmosPreviewRNG` (SplitMix64, `RandomNumberGenerator & Sendable`), `CosmosPreview` (namespace + `CosmosPreviewVariant` + `CosmosPreviewContainer`), `CosmosPreviewModifier` (`PreviewModifier`, iOS 18+, the shared-context path), `CosmosMock` + `CosmosMockWordlists`. Shared RNG state via `Mutex<CosmosPreviewRNG>` (`import Synchronization`) — never a raw `static var` mutable global (CLAUDE.md). No deprecated preview view modifiers (`.previewDevice/.previewLayout/.previewDisplayName/.previewInterfaceOrientation/.previewContext` are deprecated at OS 27); use `#Preview(_:traits:)` + `.cosmosPreviewEnv`/`.cosmosPreviewVariant`. Accessibility env keys injected via the underscore SPI (`._accessibilityReduceMotion` etc.) — get-only public, stable, used by Apple's own tooling. | Decided |
| 2026-07-17 | Split motion behavior (`CosmosConfiguration.motion`) from motion visual tokens (`CosmosTheme.motion`) | Mirrors the existing split (behavior in `CosmosConfiguration`, visual in `CosmosTheme`) — mixing `isEnabled`/policy with spring curves would conflate concerns. | `CosmosMotionConfiguration` (behavior/policy) aggregates into `CosmosConfiguration`; `CosmosMotionTokens` (springs/durations/transitions) lives in `CosmosTheme`. Both `Sendable`; both ride inside existing `@Entry` aggregates (no new `@Entry` slots). Per-instance overrides: `.cosmosMotion(_:)` (behavior), `.cosmosMotionTokens(_:)`/`.cosmosSpringStyle(_:)` (visual). | Decided |
| 2026-07-23 | Atoms impose theme defaults; `.cosmosFont`/`.cosmosForegroundStyle`/`.cosmosTint` are subtree overrides (not pass-through) | An interrupted de-styling refactor stripped theme font/foreground/tint from 3 atoms (`CosmosText`, `CosmosTextField`, `CosmosTextEditor`), which made the override modifiers **no-ops** on those atoms — an atom that does not read a token cannot be overridden via it, undercutting the 0.5.0 "Defaults + overrides SwiftUI" design. The 2026-07 organization audit (`vault/08-riscos/cosmos-org-audit-2026-07`) flagged the divergence plus the silent `.cosmosTextEditorStyle` no-op (applier deleted, modifier/enum/availability left public). | Revert the de-styling: every atom applies its theme-driven visual defaults (font/foreground/tint/motion) and reads the relevant tokens, so `.cosmosFont`/`.cosmosForegroundStyle`/`.cosmosTint`/`.cosmosControlSize` reach **every** atom. `CosmosTextEditorStyleApplier` restored (no-op resolved); `CosmosText` `.cosmosAnimation`/`.cosmosContentTransition` restored (vestigial motion event has its animated value back); `CosmosTextField` realigned with `CosmosSecureField`. Callers override per-subtree via `.cosmos*` modifiers or build a `CosmosTheme`; raw `.font`/`.foregroundStyle` remain as escape hatches. An atom is never pass-through for the tokens its override surface promises. | Decided |
